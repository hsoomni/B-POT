## ★ v2 핵심 변경 — 페이지 구성 = 웹앱 스타일
- **한 화면(스텝)씩** 보여주고, **버튼 클릭으로 다음/이전**으로 전환된다.
- **페이지 새로고침 없이** 매끄럽게 화면이 바뀐다. (스크롤로 길게 내리는 단일 페이지 ❌)
- 인트로 → 기본정보 → 질문 1~10 → 결과 → CTA 까지 **전부 앱처럼 화면 전환**.
- 각 스텝은 화면(뷰포트)에 꽉 차게, 한 번에 하나의 할 일만 보이도록.
- 진행 표시(progress)와 다음/이전 버튼이 늘 같은 자리에 있어 앱 같은 리듬을 준다.

> 구현 방향(아래 11번): 한 페이지 안에서 JavaScript가 화면을 전환하는 **SPA(단일 페이지 앱) 방식**.
> 백엔드(Django)는 마지막 제출·결과 생성·문의 저장만 담당.

## 1. 한 문장 정의
> 브랜드를 처음 만드는 사람이나 오리지널리티를 찾고 싶은 개인이, 버튼을 눌러가며 10개의 선택형 질문에 답하면 **1page 결과지(씨앗 봉투)**를 받아보는 **웹앱**.

## 2. 타깃 사용자 (두 트랙)
- **브랜드 트랙:** 스몰 브랜드 빌더, 1인 브랜드.
- **개인 트랙:** 퍼스널 브랜딩이 필요한 취준생·인플루언서 등.

## 3. 핵심 가치 / 포지셔닝
- 완성된 브랜드가 아니라 **영감·첫 씨앗(시작점)**을 주는 입문 도구.
- MBTI식 심심풀이가 아닌 **전문성 있는 유용함** → 유료 서비스 신뢰로 연결.
- 깊은 다듬기는 **커피챗·워크샵·캠페인(CTA = 회사 영역)**으로.

## 4. 핵심 기능
1. 기본 정보 입력: 개인/브랜드 트랙 선택, 이름·카테고리·타깃, 파일 첨부.
2. 10개 질문 카드: 선택형(키워드·밸런스·페르소나·비유), 트랙별 분기.
3. 결과지 출력: 6개 항목 1page, PNG 저장.
4. CTA: 커피챗·캠페인·워크샵 문의 폼 → DB + 이메일 알림.

## 5. 화면 구조 (웹앱 스텝)
순서대로 한 화면씩, 버튼으로 전환:

| # | 스텝 | 전환 |
|---|---|---|
| 0 | 인트로 (3D POT 씨앗 심기) | "시작" 버튼 → |
| 1 | 트랙 선택 | 카드 선택 → "다음" |
| 2 | 기본 정보 + 첨부 (개인정보 안내문) | "다음" |
| 3~12 | 질문 1~10 (한 화면 = 한 질문) | "다음/이전" |
| 13 | 결과지 (6항목, PNG 저장) | |
| 14 | CTA 문의 | "문의 보내기" |

- 상단 고정: 진행 바 + 현재 단계. 하단/우측 고정: 다음·이전 버튼.

## 6. 질문 카드 (요약)
- 두 트랙 각 10개. 형식 공유, 일부 분기.
- 브랜드 6항목: NAMING / MISSION / CORE VALUE / CONCEPT / PERSONA / COMMUNICATION SLOGAN
- 개인 6항목: NAMING(수식어) / MISSION / CORE VALUE / CONCEPT / PERSONA(되고 싶은 나) / **ORIGINALITY**
- NAMING·CORE VALUE·COMMUNICATION 카드에 선택적 한 줄 입력.
- 상세: (어젯밤 작성한 `B-POT_10질문카드_기획.md` 내용 유지) <!-- 편집 -->

## 7. 결과지
- 6개 항목 = 헤드라인 + 받침 2~3줄. 1분 피치는 6항목으로 대체.
- 저장: PNG (씨앗 봉투 디자인). 끝에 CTA 연결.

## 8. 디자인 시스템 (유지)
- 폰트: Pretendard. 배경: 친환경 종이(크라프트) 텍스처, 글자 검정(강조 POT).
- 팔레트: POT `#40250D` / SEED `#F2B90F` / 보조 `#F2A20C` `#BF6C06` / PAPER `#D9B88F`.
- 이모지 제외, 라인 아이콘만, 그라데이션 없음(단색 면).
- 로고: 텍스트 워드마크.
- ★ 웹앱이므로 **화면 전환 애니메이션**(슬라이드/페이드)으로 앱 느낌 강화. <!-- 편집: 전환 스타일 -->

## 9. AI / 결과 생성 (유지)
- 하이브리드: 선택지 조립(fallback) 기본, Gemini 무료 키 있으면 AI 다듬기.
- 개인정보(CTA 연락처)는 AI 미전송, DB에만 저장.

## 10. 데이터
- Result: 트랙·기본정보·첨부·답변(JSON)·생성결과·생성일.
- CTAInquiry: 유형·회사명·담당자·이메일·연결 Result·생성일.

## 11. 기술 스택 (웹앱 반영)
- 백엔드: **Django (모놀리식)**, 로그인 없음. API/제출 처리·결과 생성·문의 저장 담당.
- 프론트: **단일 페이지 + 바닐라 JS로 스텝 화면 전환**(SPA 방식). 3D는 Three.js, 결과 저장 html2canvas(PNG).
  - 제출은 페이지 이동 없이 fetch로 백엔드 호출 → 결과 화면으로 전환.
